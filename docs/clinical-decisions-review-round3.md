## What was built (1 paragraph)
A Python/FastAPI backend + React mode. The honesty spine is a **validation harness** (9 checks) that gates every item; a **per-type grader** (answer-class model); a **danger/acuity content-table** layer; and a **nano generator** whose drafts are repairedвҶ’validatedвҶ’harness-gated. v0 serves 6 hand-authored items bound to fixture ECGs. Items are graded on real tracings; vignettes are authored. The harness is the safety claim вҖ” so its soundness is everything.

## A. The harness checks вҖ” EXACT pass/fail logic (audit these)
Each runs `check(item, packet, prior_packet)`; a hard-stop failure blocks the item. `packet` exposes `supported_objectives` (curation Tier-A/B concept set), `features` (heart_rate, pr_ms, qrs_ms, qtc_ms, axis_deg, per-lead STвҖҰ), and `rois` (`{lead, concept, timeStartSec, timeEndSec, ampMinMv, ampMaxMv}`).

1. **compound_option_parsing** вҖ” parses each option's text into `{get_more_data, safety_action_present, safety_tokens, action_delayed}` by keyword cues. Fails only if an option is tagged `insufficient_data` AND has a parallel safety action AND not delayed (the corrected Case W rule). *Everything else is advisory.*
2. **evidence_binding** вҖ” per `ecg_supports` claim `{objective_id, threshold?, leads?, source_type}`: (a) if source_type вҲҲ {measured, curated_label} then `objective_id` must вҲҲ `supported_objectives`; (b) source_type `authored_context` inside ecg_supports вҶ’ fail; (c) if `threshold` given, it must parse as `feature<cmp>value` and evaluate TRUE against `features` (else fail); a `measured` claim with no threshold вҶ’ fail; (d) if `leads` given, at least one must have a grounded ROI for that concept (via a conceptвҶ’ROI map, with the concept's own name also accepted). Plus: a fixed list of territory adjectives ("inferolateral", "reciprocal", "territorial", "anterolateral", "anteroseptal infarct", "inferoposterior") in the stem/option text вҶ’ fail unless some supported MI/ST concept has a grounded territory ROI.
3. **acute_temporal_causality** вҖ” if the stem+stem_adds contain any of a fixed cue list ("acute onset", "acutely", "escalating", "evolving", "crushing", "diaphoresis", "rising troponin", "new-onset", "hyperacute", "worsening", "sudden onset", "minutes ago", вҖҰ) вҶ’ fail UNLESS `prior_ecg_id` is present OR the manifest blob contains a token in {serial, prior comparison, acute_dataset, staff_iii, telemetry}.
4. **symptom_causality** вҖ” uses a per-concept table mapping symptomвҶ’{may_explain, may_contextualize, must_not_explain}. Fails iff: the item's `chips.symptom` is `must_not_explain` for ALL relevant supported concepts AND none `may_explain` it AND some ideal/acceptable option's text contains an escalation cue {admit, pacing, transvenous, urgent, activate, cardiology, icu, escalat}.
5. **transient_event_evidence** вҖ” cues {torsades, "runs of", "nonsustained vt", pause, "telemetry showed", "monitor showed", captured, вҖҰ} in stem+stem_adds вҶ’ fail unless `tested_scope == rhythm_only` OR blob has {rhythm_strip, telemetry, authored_data_layer}.
6. **required_safety_action** вҖ” for each supported concept in a table (third-degree AV block вҶ’ [pacing_pads, tcp_ready, call_help, bedside_now]; Mobitz II вҶ’ [tcp_ready, call_help, bedside_now]; wide-complex-tachy вҶ’ [synchronized_cardioversion, defib_pads, act_now]), every ideal/acceptable option must carry вүҘ1 of those tokens (from its `required_safety_tokens` or text-parsed tokens), else fail.
7. **acuity_cap_by_concept** вҖ” `effective_tier = max(base_acuity[c] for c in supported)` (+1 per measurement-bump rule: AFib/flutter/SVT & HRвүҘ150; any HRвүӨ40; QTcвүҘ500). Fails iff `item.acuity_tier` rank > effective_tier rank. (Tier-vs-tier; the per-concept *action-urgency* caps are NOT enforced on individual options вҖ” see task 1.)
8. **scoring_schema** вҖ” click: must have `roi_target`, no answer-class options. oldnew: ternary values present; an option keyed `old/unchanged` as ideal/acceptable REQUIRES `display_spec.mode==stacked_twelve_lead` + `prior_ecg_id`. triage: warns if an option scores an `ecg_recognition` axis.
9. **display_checks** вҖ” `tested_scope==rhythm_only` fails if any supported objective is outside a rhythm-OK set (axis/territory/BBB can't be shown on a strip); oldnew requires stacked display.

## B. The generator REPAIR step (audit for laundering)
A cheap model's draft is normalized before validation: lowercase/snap enums (situation synonyms, tested_scope/display-mode вҶ’ default `full_12_lead`/`twelve_lead` if unknown); coerce `chips.age`вҶ’int, drop placeholder chip strings, snap `symptom` to the known vocab (else null); **prune any keys not in the allowed schema** for EvidenceClaim/manifest/Option/display_spec; strip a trailing "?" copied from optional-field docs; hoist a misplaced `epistemic_status`; wrap a scalar `leads`вҶ’list; **drop a non-string `threshold`**; and **downgrade `source_type:"measured"` with no threshold вҶ’ `"curated_label"`**. Only then validate + run the harness; a pass вҶ’ status `harness_pass` (not auto-served).

## C. The grader math (audit for farming)
- Base credit by class: ideal 1.0, acceptable 0.8, over_triage_safe 0.6, under_triage 0.3, insufficient_data 0.3, unsafe 0.0. `correct := base вүҘ 0.6`.
- Confidence (1вҖ“5 вҶ’ lowвүӨ2 / med=3 / highвүҘ4) multiplies: correct Г— {high 1.0, med 0.85, low 0.65}; wrong Г— {high 0.0, med 0.5, low 1.0}. (wrong@high вҶ’ 0 + flag.)
- Grade-time caps: ideal/acceptable missing a required safety token вҶ’ base capped to 0.3 (+flag). `insufficient_data` on a `determined` item вҶ’ capped to 0.40; on an `intentionally_underdetermined` item вҶ’ raised to 1.0.
- Calibration label from the shift's events: confident-but-brittle if high-conf-wrong вүҘ 30% of items; else risky-under-caller / cautious-over-caller if under/over вүҘ 30%; else well-calibrated.
